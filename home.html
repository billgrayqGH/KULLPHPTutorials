PHP (Hypertext Preprocessor)

What is PHP?

It is a server scripting language which is used to control user-access. It establishes interaction between the users computer or the client with the Web Server. It also collects form data from html pages. Sends and receives cookies. You can create, delete, open, close, read and write files on a data server.



What are Cookies?

A cookie is often used to identify a user. A cookie is a small file that the server adds on the user's computer. Each time the same computer requests a page with a browser, it will send the cookie too. With PHP, you can both create and retrieve cookie values. : cookies are used in forms and other web pages which require you to enter information, with cookies if the user happens to visit the same website the cookie helps to save time in entering the required information over and over again.



Myths about Cookies

MYTH
- Cookies are like worms/viruses and can erase user files on their computer.
- Cookies are a form of spyware and collects your personal information.
- Cookies generate popups and spam and used for ads on a website.




FACTS
- Cookies are only data that is sent by the web server, not a program code.
- Cookies cannot erase or read information from the user's computer.
- Cookies CAN be used to track your viewing habits on a particular site.

How to code PHP Cookies?

Unlike CSS and JavaScript, PHP can be run separately from html or embedded to html. It is coded using the tag element <?php and ends with ?>. Since PHP is server-side scripting it needs a web server before running the code on your web browser. The keyword setcookie() is used to create a cookie and must be written before the html tag.

Syntax for setcookie() - setcookie(name,value,expire,path,domain,secure,httponly);

    Parameter and description
  • name - assigns the name of the cookie(Required)
  • value - assigns the value of the cookie(Optional)
  • expire - specifies when will the cookie expires. "time()" means the time of creation, "+86400" equals to one day (Optional)
  • path - determines the server path for the cookie. "/" means the cookie is available on the entire domain.
    /php/ means the cookie is available only on the php directory.(Optional)
  • domain - specifies the domain for the cookie.(Optional)
  • secure - describes if the cookie should be transmitted on secure HTTPS connection. Default value is FALSE.(Optional)
  • httponly - describes if the cookie can only be accessible through http protocol only meaning the cookie cannot be modified by scripting language. Default value is FALSE. (Optional)

PHP Code

	
	<?php
	$cookie_name = "user";
	$cookie_value = "Cookie Monster";
		
	setcookie($cookie_name, $cookie_value, time(); + (86400 * 30), "/");
	// 86400 = 1 day


	if(!isset($_COOKIE[$cookie_name])) {
		echo "Cookie named '" . $cookie_name . "' is not created!";
		} 
	else {
		echo "Cookie '" . $cookie_name . "' is created!<br>";
		echo "Value is: " . $_COOKIE[$cookie_name];
		}
	?>
